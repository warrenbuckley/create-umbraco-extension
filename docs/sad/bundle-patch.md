# bundle.manifests.ts Patch Logic

Every Umbraco extension project has a central `src/bundle.manifests.ts` that Vite uses as its entry point. All extension manifests must be imported and spread into this file — this is how Umbraco discovers them at runtime.

## Why patching is needed

When a user adds a second (or third) extension to an existing project, the CLI needs to register the new extension in `bundle.manifests.ts` without breaking existing imports.

## The patch

The update is done with **simple string manipulation — no AST parser**. The file structure is predictable enough that regex-based insertion is safe and maintainable.

**Before** (existing bundle):
```typescript
import { manifests as entrypoints } from './entrypoints/manifest.js';
import { manifests as myDashboard } from './extensions/my-dashboard/manifests.js';

export const manifests: Array<UmbExtensionManifest> = [
  ...entrypoints,
  ...myDashboard,
];
```

**After** (CLI adds an entity action):
```typescript
import { manifests as entrypoints } from './entrypoints/manifest.js';
import { manifests as myDashboard } from './extensions/my-dashboard/manifests.js';
import { manifests as deleteAction } from './extensions/delete-action/manifests.js';

export const manifests: Array<UmbExtensionManifest> = [
  ...entrypoints,
  ...myDashboard,
  ...deleteAction,
];
```

## Algorithm

1. Append the new `import` line at the end of the import block (detected by the last line starting with `import`)
2. Insert the new `...spread` entry before the closing `]` of the manifests array
3. **Idempotency check** — if the import path is already present, skip without modifying

The patch logic lives in `src/utils/write-files.ts` as an exported `patchBundle()` function, making it independently testable.

## Why no AST parser?

The `bundle.manifests.ts` file has a fixed, predictable structure — it's generated by this same CLI. An AST parser (e.g. TypeScript compiler API) would add significant complexity and a heavyweight dependency for a problem that doesn't need it. String manipulation is simpler, faster, and easier to test.
